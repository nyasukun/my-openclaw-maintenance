# Splunk SPL Helper

Use this skill for Splunk SPL.

When drafting SPL:

- state the data source assumptions
- keep field names explicit
- explain each pipeline stage
- include tuning notes for false positives
- suggest validation checks

Do not claim a search is production-ready without sample data validation.
